# Todos API

The Todos API provides comprehensive task management with projects, subtasks, priorities, labels, and smart search capabilities.

## Data Model

### Todo Object

<ResponseField name="id" type="string">
  Unique identifier for the todo
</ResponseField>

<ResponseField name="user_id" type="string">
  User ID who owns the todo
</ResponseField>

<ResponseField name="title" type="string">
  Todo title (1-200 characters)
</ResponseField>

<ResponseField name="description" type="string">
  Optional description (max 2000 characters)
</ResponseField>

<ResponseField name="priority" type="string">
  Priority level: `high`, `medium`, `low`, or `none`
</ResponseField>

<ResponseField name="completed" type="boolean">
  Whether the todo is completed
</ResponseField>

<ResponseField name="due_date" type="string">
  ISO 8601 due date timestamp (optional)
</ResponseField>

<ResponseField name="due_date_timezone" type="string">
  Timezone for the due date (e.g., "America/New_York")
</ResponseField>

<ResponseField name="labels" type="array">
  Array of label strings for categorization (max 10)
</ResponseField>

<ResponseField name="project_id" type="string">
  Project ID the todo belongs to (defaults to Inbox)
</ResponseField>

<ResponseField name="subtasks" type="array">
  Array of subtask objects with `id`, `title`, and `completed` fields
</ResponseField>

<ResponseField name="workflow_id" type="string">
  Associated workflow ID (optional)
</ResponseField>

<ResponseField name="workflow_categories" type="array">
  Tool categories from linked workflow steps
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 creation timestamp
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 last update timestamp
</ResponseField>

<ResponseField name="completed_at" type="string">
  ISO 8601 completion timestamp (if completed)
</ResponseField>

## Endpoints

### List Todos

Get todos with filtering, search, and pagination.

```bash
GET /api/v1/todos
```

<ParamField query="q" type="string">
  Search query (supports text, semantic, and hybrid search)
</ParamField>

<ParamField query="mode" type="string" default="hybrid">
  Search mode: `text`, `semantic`, or `hybrid`
</ParamField>

<ParamField query="project_id" type="string">
  Filter by project ID
</ParamField>

<ParamField query="completed" type="boolean">
  Filter by completion status
</ParamField>

<ParamField query="priority" type="string">
  Filter by priority: `high`, `medium`, `low`, or `none`
</ParamField>

<ParamField query="due_today" type="boolean">
  Show only todos due today
</ParamField>

<ParamField query="due_this_week" type="boolean">
  Show only todos due this week
</ParamField>

<ParamField query="overdue" type="boolean">
  Show only overdue todos
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number (1-based)
</ParamField>

<ParamField query="per_page" type="integer" default="50">
  Items per page (max: 100)
</ParamField>

<ParamField query="include_stats" type="boolean" default="false">
  Include statistics in response
</ParamField>


### Create Todo

Create a new todo item.

```bash
POST /api/v1/todos
```

```json Request
{
  "title": "Complete API integration",
  "description": "Integrate GAIA API into application",
  "priority": "high",
  "due_date": "2026-02-25T18:00:00Z",
  "due_date_timezone": "America/New_York",
  "labels": ["api", "development"],
  "project_id": "proj_123"
}
```

### Get Todo

Retrieve a specific todo by ID.

```bash
GET /api/v1/todos/{todo_id}
```

### Update Todo

Update a todo (partial updates supported).

```bash
PUT /api/v1/todos/{todo_id}
```

```json Request
{
  "completed": true,
  "priority": "medium"
}
```

### Delete Todo

Delete a todo.

```bash
DELETE /api/v1/todos/{todo_id}
```

## Subtasks

### Create Subtask

Add a subtask to a todo.

```bash
POST /api/v1/todos/{todo_id}/subtasks
```

```json Request
{
  "title": "Write integration tests"
}
```

### Update Subtask

Update a subtask.

```bash
PUT /api/v1/todos/{todo_id}/subtasks/{subtask_id}
```

```json Request
{
  "title": "Write comprehensive tests",
  "completed": true
}
```

### Delete Subtask

Delete a subtask.

```bash
DELETE /api/v1/todos/{todo_id}/subtasks/{subtask_id}
```

### Toggle Subtask

Toggle subtask completion status.

```bash
POST /api/v1/todos/{todo_id}/subtasks/{subtask_id}/toggle
```

## Bulk Operations

### Bulk Update

Update multiple todos at once.

```bash
PUT /api/v1/todos/bulk
```

```json Request
{
  "todo_ids": ["id1", "id2", "id3"],
  "updates": {
    "completed": true,
    "priority": "high"
  }
}
```

### Bulk Delete

Delete multiple todos.

```bash
DELETE /api/v1/todos/bulk
```

```json Request
["id1", "id2", "id3"]
```

### Bulk Complete

Mark multiple todos as completed.

```bash
POST /api/v1/todos/bulk/complete
```

```json Request
["id1", "id2", "id3"]
```

```bash
POST /api/v1/projects
```


## Workflows

### Generate Workflow

Generate a workflow for a todo.

```bash
POST /api/v1/todos/{todo_id}/workflow
```

Returns immediately with `generating` status. Listen for `workflow.generated` WebSocket event.

```json Response
{
  "status": "generating",
  "todo_id": "todo_123",
  "message": "Workflow generation started. Listen for 'workflow.generated' WebSocket event."
}
```

### Get Workflow Status

Get workflow generation status.

```bash
GET /api/v1/todos/{todo_id}/workflow-status
```

```json Response
{
  "todo_id": "todo_123",
  "has_workflow": true,
  "is_generating": false,
  "workflow_status": "completed",
  "workflow": {...}
}
```

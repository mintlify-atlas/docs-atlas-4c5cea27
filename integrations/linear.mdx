The Linear integration enables GAIA to create, update, and manage issues in your Linear workspace. Let AI handle issue tracking, project management, and development workflows naturally.

## Features

- **Issue Management**: Create, update, and track Linear issues
- **Project Organization**: Organize issues across projects and teams
- **Label & Status**: Manage issue labels, priorities, and statuses
- **Assignment**: Assign issues to team members
- **Cycle Management**: Work with Linear cycles and sprints
- **Comment & Collaborate**: Add comments and collaborate on issues
- **Search & Filter**: Find issues with advanced filtering

## Setup

<Steps>
  <Step title="Navigate to Integrations">
    Go to **Settings > Integrations** in GAIA
  </Step>
  <Step title="Find Linear">
    Search for "Linear" in the Developer category
  </Step>
  <Step title="Connect Account">
    Click **Connect** to start OAuth flow
  </Step>
  <Step title="Authorize Access">
    Sign in to Linear and authorize GAIA to access your workspace
  </Step>
  <Step title="Select Workspace">
    Choose which Linear workspace to connect (if you have multiple)
  </Step>
</Steps>

<Warning>
  Make sure you have appropriate permissions in your Linear workspace to create and modify issues.
</Warning>

## OAuth Configuration

The Linear integration uses OAuth 2.0:

- **Provider**: Composio
- **Toolkit**: LINEAR
- **Auth Type**: OAuth 2.0
- **Managed By**: Composio platform

## Available Actions

The Linear agent can perform:

- Create issues with title, description, and properties
- Update issue status, priority, assignee
- Add labels and tags
- Manage issue relationships (blocks, blocked by)
- Add comments to issues
- Create and manage projects
- Organize issues in cycles
- Search and filter issues
- Get current user information
- Manage team assignments
- Track issue history

## Workflows & Triggers

Linear supports powerful automation triggers:

### Issue Created Trigger

**Trigger**: When a new issue is created

**Configuration**:
```yaml
trigger: linear_issue_created
config:
  team_id: ""  # Optional: filter by team
```

**Use Cases**:
- Notify team in Slack
- Create corresponding Notion page
- Auto-assign based on labels
- Log to analytics

### Issue Updated Trigger

**Trigger**: When an issue is updated

**Configuration**:
```yaml
trigger: linear_issue_updated
config:
  team_id: ""  # Optional: filter by team
```

**Use Cases**:
- Track status changes
- Send progress updates
- Update external systems
- Notify stakeholders

### Comment Trigger

**Trigger**: When a comment is added to an issue

**Configuration**:
```yaml
trigger: linear_comment_added
config:
  team_id: ""  # Optional: filter by team
```

**Use Cases**:
- Notify issue assignee
- Forward important comments to Slack
- Track team discussions

## AI Agent Details

Linear uses a specialized subagent:

- **Agent Name**: `linear_agent`
- **Domain**: Issue tracking and project management
- **Capabilities**:
  - Creating well-structured issues
  - Understanding project context
  - Breaking down complex tasks
  - Managing issue workflows
  - Sprint and cycle planning
  - Team coordination

### Agent Intelligence

The Linear agent can:
- Infer issue priority from description
- Suggest appropriate labels
- Break down epics into smaller tasks
- Understand team workflows
- Recommend assignees based on expertise
- Detect related issues

## API Reference

Key tools available:

| Tool | Description |
|------|-------------|
| `LINEAR_GET_CURRENT_USER` | Get authenticated user info |
| `LINEAR_CREATE_ISSUE` | Create a new issue |
| `LINEAR_UPDATE_ISSUE` | Update issue properties |
| `LINEAR_GET_ISSUE` | Get issue details |
| `LINEAR_SEARCH_ISSUES` | Search with filters |
| `LINEAR_CREATE_COMMENT` | Add comment to issue |
| `LINEAR_LIST_TEAMS` | Get workspace teams |
| `LINEAR_LIST_PROJECTS` | Get projects |
| `LINEAR_GET_CYCLE` | Get cycle information |
| `LINEAR_CREATE_PROJECT` | Create new project |
| `LINEAR_UPDATE_PROJECT` | Update project details |

For complete API documentation, see [Linear Tools Reference](/api-reference/integrations/linear).

## Metadata Extraction

GAIA automatically extracts your Linear user info:

```json
{
  "user_id": "user_abc123",
  "username": "John Smith",
  "email": "john@company.com"
}
```

This enables:
- Auto-assignment of issues
- Filtering by current user
- Team mentions and notifications

## Troubleshooting

<AccordionGroup>
  <Accordion title="Can't create issues">
    Verify you have write permissions in your Linear workspace. Check with your workspace admin.
  </Accordion>
  
  <Accordion title="Team not found">
    Make sure you're referencing the correct team name or ID. Use `List Teams` to see available teams.
  </Accordion>
  
  <Accordion title="Status update failed">
    The status must be valid for your team's workflow. Each team can have custom status values.
  </Accordion>
  
  <Accordion title="Assignment errors">
    You can only assign issues to team members. Verify the user is part of the target team.
  </Accordion>
</AccordionGroup>

## Best Practices

<Tip>
  Use clear, descriptive issue titles. Good: "Login API returns 500 on invalid password". Bad: "Fix login".
</Tip>

<Tip>
  Include acceptance criteria in issue descriptions to help GAIA understand completion requirements.
</Tip>

<Warning>
  When bulk updating issues, always specify team ID or project to avoid ambiguity.
</Warning>

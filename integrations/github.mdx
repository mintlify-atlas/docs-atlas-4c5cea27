The GitHub integration allows GAIA to interact with your repositories, manage issues, review pull requests, and automate development workflows. Let AI assist with your entire software development lifecycle.

## Features

- **Repository Management**: Create, update, and manage repositories
- **Issue Tracking**: Create, update, and search issues
- **Pull Request Management**: Review, comment, and merge PRs
- **Branch Operations**: Create and manage branches
- **Code Review**: Comment on code and suggest changes
- **Actions & CI/CD**: Monitor GitHub Actions workflows
- **Collaboration**: Manage collaborators and teams
- **Release Management**: Create and manage releases

## Setup

<Steps>
  <Step title="Open Integrations">
    Navigate to **Settings > Integrations** in GAIA
  </Step>
  <Step title="Find GitHub">
    Search for "GitHub" in the Developer category
  </Step>
  <Step title="Connect Account">
    Click **Connect** to start OAuth flow
  </Step>
  <Step title="Authorize Access">
    Sign in to GitHub and authorize GAIA
  </Step>
  <Step title="Select Repositories">
    Choose which repositories GAIA can access (or select "All repositories")
  </Step>
  <Step title="Complete Setup">
    Click "Authorize" to complete the connection
  </Step>
</Steps>

<Warning>
  GitHub requires you to explicitly grant access to repositories. You can modify repository access anytime in GitHub Settings > Applications.
</Warning>

## OAuth Configuration

The GitHub integration uses OAuth 2.0:

- **Provider**: Composio

- **Auth Type**: OAuth 2.0
- **Featured**: Yes
- **Managed By**: Composio platform

## Workflows & Triggers

GitHub supports comprehensive automation triggers:

### Commit Event Trigger

**Trigger**: When a new commit is pushed

**Configuration**:
```yaml
trigger: github_commit_event
config:
  owner: "username"
  repo: "myrepo"
```

**Use Cases**:
- Notify team of commits
- Trigger builds
- Update project tracking
- Log commit history

### Pull Request Event Trigger

**Trigger**: When a PR is opened, closed, or synchronized

**Configuration**:
```yaml
trigger: github_pr_event
config:
  owner: "username"
  repo: "myrepo"
```

**Use Cases**:
- Auto-review PRs
- Notify reviewers
- Update Linear issues
- Post to Slack

### Star Added Trigger

**Trigger**: When someone stars your repository

**Configuration**:
```yaml
trigger: github_star_added
config:
  owner: "username"
  repo: "myrepo"
```

**Use Cases**:
- Thank new stargazers
- Track repository popularity
- Send notifications

### Issue Added Trigger

**Trigger**: When a new issue is created

**Configuration**:
```yaml
trigger: github_issue_added
config:
  owner: "username"
  repo: "myrepo"
```

**Use Cases**:
- Auto-label issues
- Notify maintainers
- Create Linear tasks
- Triage automation

## API Reference

Key tools available:

| Tool | Description |
|------|-------------|
| `GITHUB_GET_THE_AUTHENTICATED_USER` | Get user info |
| `GITHUB_CREATE_AN_ISSUE` | Create new issue |
| `GITHUB_UPDATE_AN_ISSUE` | Update issue |
| `GITHUB_LIST_REPOSITORY_ISSUES` | List issues |
| `GITHUB_CREATE_A_PULL_REQUEST` | Create PR |
| `GITHUB_LIST_PULL_REQUESTS` | List PRs |
| `GITHUB_MERGE_A_PULL_REQUEST` | Merge PR |

| `GITHUB_CREATE_A_RELEASE` | Create release |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Repository access denied">
    Check that you granted GAIA access to the specific repository during OAuth. Update permissions in GitHub Settings > Applications.
  </Accordion>
  
  <Accordion title="Can't merge pull request">
    Verify you have merge permissions and all required checks have passed. Some repositories require admin approval.
  </Accordion>
  
  <Accordion title="Actions workflow not triggering">
    Ensure the workflow file is properly configured and the event trigger matches your setup.
  </Accordion>
  
  <Accordion title="Rate limit errors">
    GitHub has API rate limits (5,000 requests/hour for authenticated users). GAIA handles rate limiting automatically.
  </Accordion>
</AccordionGroup>
